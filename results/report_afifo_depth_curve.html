案例：16×16 双向为何在深 AFIFO 后 makespan 骤降至 244？

表中可见：cap≤40 时 makespan 稳定在 379 cy（AFIFO 仅需 3）； cap≥45 时降至 244 cy（约 −36%）。本 sweep 在 40 与 45 之间未采样， 故断崖首次出现在 cap=45——真实门槛是下面这套 spread=0 调度的 AFIFO 峰值深度 41。

1. 为什么会下降？——两种调度范式切换

cap≤40 时的最优方案来自 schedule_atomic（相位错开 / pacing）：

  • 每个源作为整体原子放置；若某次跨界会使任意边界 AFIFO 在任意 cycle 超过 cap，就整体推迟该源的注入。
  • 在 AFIFO 预算很小（深度≤3）时，只能大量「等」在边界，注入偏移累积 → makespan ≈379 cy，但 AFIFO 峰值仅 3。
  • 这是保守、低缓冲策略：用更长时间换更浅的 AFIFO。

cap≥41 时，另一类方案变得可行：schedule 的 spread=0（环上链路时分插入）：

  • 四象限 Hamilton 环先按固有节拍跑（Pass1 本象限 home 子树）；跨界 flit 插入环内链路空闲 send 槽（Pass2），允许在 AFIFO 中短暂排队。
  • 不强制 per-source 原子 pacing，跨界 burst 与环内 conveyer 更自然对齐 → makespan 244 cy（约 −36%）。
  • 代价：单链路 AFIFO 峰值约 41（8 路并行均衡后约 36），必须用更深的边界 FIFO 承受等待。

结论：下降不是「缓冲越深传输越快」的连续渐变，而是低深度下只能用 atomic 保守调度；深度够深后 spread=0 的环时分调度才合法并显著更优。

2. 为什么门槛在 41（而不是 40 或 3）？

对每个候选调度，我们要求 afifo_depth ≤ cap（单链路峰值）。

  • spread=0 方案的实测峰值 AFIFO = 41。
  • cap<41 时该方案不被允许，搜索只能在 atomic 等低深度方案里选 → 仍约 379 cy。
  • cap≥41 时 spread=0 首次入选候选集，244 < 379，曲线断崖式下探（本 sweep 首个满足的采样点为 cap=45）。

cap=3 时虽也允许「深度≤3」，但 spread=0（峰值 41）仍不可行；atomic 在 cap=3 已到其自身最优 379，故 cap=3~40 平台不变。

3. 与 eject 下界的关系

16×16 双向 eject 下界 = ⌈(N−1)/2⌉ = 128 cy。244 cy 约为下界的 1.91×，仍远高于下界——瓶颈在跨界+环内链路时分与 AFIFO 等待，而非下 ramp 带宽。

cap≤40 最优：atomic/natural，mk=379，AFIFO峰值=3。 cap≥45 最优：schedule spread=0，mk=244，AFIFO峰值=41，均衡深度≈36。

对角象限均衡：半经水平邻居、半经垂直邻居

更新：2026-06-24T08:37:21.467183+00:00 · results/border_bal_afifo_depth_sweep.json · sched_ring_zerobuf.deliv_border_bal_quads

路由改动

原始 border 短弧把整个对角象限（QD，如左上源的 64 个目的点）全部经 水平相邻象限 QH 投递（QD 每一列都从中线 y-border 进入）。 均衡方案把 QD 的 64 个目的点对半拆分：

  • 上半 32 个（QD 上半行）经 QH：从中线 y-border 进入，沿列向下短弧。
  • 下半 32 个（QD 下半行）经 垂直相邻象限 QV：沿 QV 列向下后跨中线 x-border 进入 QD，沿行横向短弧。

这样原本只承载 QH 流量的中线 x-border 也分担一半对角流量，两条中线边界负载更对称。

结论

  • AFIFO 峰值下降：在浅缓冲（atomic）区间，均衡把单链路 AFIFO 峰值显著降低。 16×16 双向 cap=5 时：原始需 afifo=5（mk=404），均衡只需 afifo=2（mk=423）——峰值 −60%。 8×8 单向 cap≥2：3→2；16×16 双向 cap=3~4：3→2。
  • makespan 略升：对角下半经 QV 要多走垂直象限的整列短弧，关键路径变长，双向配置约 +5% （16×16 双向最优 244→267，8×8 双向 89→93）。跨界时延=10cy 放大了这一代价。
  • 本质是权衡：把对角流量摊到两条中线边界，用更浅的 AFIFO（更小的边界 FIFO 面积）换取约 5% 的 makespan。 若 AFIFO 深度是受限/昂贵资源（如 GALS 边界异步 FIFO），均衡方案更优；若追求最短 makespan 且 FIFO 充足，原始单路径更快。

Router 每 port 缓冲 K × 边界 AFIFO 深度 A

更新：2026-06-24T08:37:50.063952+00:00 · results/router_afifo_depth_sweep.json · 生成 sweep_router_afifo_depth.py

在现有「router 零 buffer（K=0）」曲线基础上，允许每个 router 输出 port / 下 ramp port 峰值排队深度 ≤ K（flit），同时边界 AFIFO 单链路峰值 ≤ A。 候选调度：strict（spread/atomic，K=0 时与上图相同）+ pipelined TDM（环内链路可短暂排队）。

机制要点

  • K=0：与上文「router 零 buffer」完全相同；等待只能发生在边界 AFIFO（或源 PE 注入偏移）。
  • K≥1：环内 Hamilton 链路可「就绪但链路忙」时在 router 输出 port 排队；下 ramp 同理（eject_buf）。 pipelined 典型需求 ring_buf≈1、eject_buf≈2~3 → 往往需 K≥2（16×16 单向 pipelined 需 K≥3）。
  • AFIFO 与 router buffer 解耦：跨界等待仍在 AFIFO；加深 K 主要缓解象限内环与eject 对齐，不能替代深 AFIFO 对 spread=0 大跨界调度的需求。
