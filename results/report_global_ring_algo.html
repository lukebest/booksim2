全局 Hamilton 环 Allgather：跨 message size 的最优 makespan 生成算法

16×16 mesh，N=256。 H=4 cy / V=6 cy；跨 reticle 边界链路固定 6 cy（无 AFIFO 等待）；router_buf=0，刚性 offset pack。 数值由 sched_zerobuf_compare.pack 复算。

1. 模型与问题

在 16×16 mesh 上用单条全局 Hamilton 环（Q=1）做 allgather。 本模型为刚性 0-buffer pack（与 sched_zerobuf_compare 一致）： router 零缓冲、链路无排队；跨 reticle 边界链路仅增加固定 6 cycle 传播延迟， 不在 AFIFO 中等待。 对 wormhole 报文 \(m\) flit，求最小 makespan 及最优环走法。

定义（输入）。 mesh \(G=(V,E)\)，\(|V|=N\)；水平链路时延 \(H\)、垂直时延 \(V_{\!}\)； reticle 划分 \(\mathcal R\)（16×16 时为四个 8×8 象限）； 跨 reticle 物理链路经 AFIFO 接口，传播时延固定为 \(L_x\)（本例 \(L_x=6\) cy）， 不允许在 AFIFO 中排队等待； 报文长度 \(m\) flit（wormhole）；下 ramp 带宽 \(\rho\) flit/cy/node。
\[ \ell(u,v)=\begin{cases} L_x & \mathrm{ret}(u)\neq\mathrm{ret}(v)\ \text{（跨 reticle，固定延迟）}\\ H & \text{同 reticle，水平}\\ V_{\!} & \text{同 reticle，垂直}\end{cases} \]

刚性 0-buffer 模型。 每个源 \(s\) 的投递结构（沿 Hamilton 环转发）一旦确定， 各链路与 eject 的相对时刻即固定；唯一可调参数是源侧注入偏移 \(\mathit{inject}_s\) （数据暂存于 PE/SRAM，非 router 缓冲）。 滑动 \(\mathit{inject}_s\) 使全局日历满足：

\[ \begin{aligned} &\text{(L)}\ \forall e,t:\ \text{链路 }e\text{ 在 }[t,t{+}m)\ \text{至多 1 flit},\\ &\text{(R)}\ \forall c,t:\ \text{节点 }c\text{ 下 ramp 在 }[t,t{+}m)\ \text{至多 }\rho\text{ flit},\\ &\text{(C)}\ \forall c:\ \text{eject@}c = (N-1)\,m . \end{aligned} \]

全局 Hamilton 环 \(\pi=(v_0,\ldots,v_{N-1})\) 闭合且相邻为 mesh 邻居。 源 \(s\) 在环上注入后沿 \(\pi\) 逐 hop 转发；hop 延迟累加 \(\ell(\cdot,\cdot)\)。 makespan 为最晚 eject 完成时刻。

\[ \boxed{\ \mathrm{MK}^\star(m)=\min\Big\{\min_{\pi,\,dir,\,\sigma}\mathrm{MK}(\pi,\sigma),\ \ m\cdot \mathrm{MK}^\star(1)\Big\}\ } \qquad \mathrm{MK}^\star(m)\ge \Big\lceil \tfrac{(N-1)m}{\rho}\Big\rceil \]

3. 形式化算法

# 刚性 pack —— 跨 message size 最优 makespan
def best_makespan(G, H, V, Lx, m, ramp):
    Cands ← { rect, vflip, vband, ... }           # 合法 Hamilton 环
    best ← ∞
    for order in Cands:
        foot ← { s : fp_ring(order, s, dir) for s ∈ V }   # hop 用 ℓ(u,v)
        for src_order in {natural, corner, ring, ...}:
            mk ← pack(foot, ramp, src_order, flits=m)   # 滑动 inject_s
            best ← min(best, mk)
    return min(best, m × best_makespan(..., m=1, ...))

实现：utils/sched_zerobuf_compare.py 的 fp_ring + pack； 跨区 hop 用 sim_fused_rings.link_lat（cross=6）。 本页： utils/gen_global_ring_algo.py。

4. 自然语言算法

  1. 候选 Hamilton 环。 在 16×16 上生成合法闭合环（见 §2 示意图）： rect、vflip、vband 等；验证相邻节点为 mesh 邻居。
  2. 刚性 footprint。 对每个源 \(s\)，按选定环顺序构造 fp_ring：记录每条有向链路 \((p\!\to\!c)\) 的发送时刻与每个节点的 eject 时刻； 相邻 hop 间隔 \(\ell(p,c)\)——同 reticle 内为 \(H\) 或 \(V_{\!}\)，跨 reticle 为固定 \(L_x=6\)。 中间 router 不缓冲，不存在 AFIFO 排队。
  3. offset pack。 按源序逐个放置 footprint：为源 \(s\) 找最小非负 \(\mathit{inject}_s\)， 使新增 footprint 不与已有链路/下 ramp 日历冲突（wormhole 占连续 \(m\) 周期）。 冲突则 \(\mathit{inject}_s\mathrel{+}=1\)。 多试几种源序（natural / corner / ring …），取最小 makespan。
  4. 跨 message size。 对每个 \(m\)，重复 pack（flits=\(m\)）； 再与 \(m\cdot\mathrm{MK}^\star(1)\)（串行重放 \(m\) 次 \(m{=}1\) 最优 allgather）取小。
  5. 输出。 返回 \(\mathrm{MK}^\star(m)\) 及对应的 \((\pi, dir, src\_order)\)。

形状选择规律（16×16，\(L_x=6\)）

  • 不同 Hamilton 走法改变环上各链路的负载与跨 reticle hop 的时空分布；pack 在无 AFIFO 等待约束下 必须为跨区 hop 留出刚性时间窗，故形状直接影响可行性与 makespan。
  • vband 与 rect/vflip 的跨区 hop 数量与位置不同， 在 \(m\) 较大时可能导致不同的 pack 紧密度（见 §4 数值表）。
  • ramp=2 时双向环（bi）通常优于单向（uni），因 eject 带宽翻倍。
