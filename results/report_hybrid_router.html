Hybrid TDM + Packet-Switched Router

16×16 mesh（N=256），32nm ITRS（Orion 模型），W=128 bit/flit。 生成脚本 utils/gen_hybrid_router_report.py。

1. 设计目标

NoC 需同时支持两类流量模式：

  • 时隙表 TDM：离线 calendar 预配置，无冲突、无阻塞、无 router buffer（broadcast / allgather / reduce 等集合通信）。
  • 分组交换 packet：statistical multiplexing + VC buffer，支持 all-to-all / any-to-any 等未知目的地流量。

本节给出三种 router 微架构（TDM-only / packet-only / hybrid），评估面积与功耗，并分析异构 NoC 可行性。

2. 模型参数

拓扑16×16 mesh，5-port router（4 neighbor + local ramp）。

TDMnum_vc=1，skid depth=2；slot-table depth=255 cy（allgather M=1 period）。

Packetnum_vc=16，depth=8；iSLIP VC+SW allocator（面积估算）。

Tech32nm，Vdd=0.9V，fCLK=9.71GHz；Orion formulas from src/power/power_module.cpp + src/power/techfile.txt。

Activitydynamic power at α=0.15 average port utilization.

3. 微架构设计

3.1 TDM / calendar router

  • 每 output port 独立 slot-table SRAM（depth=255）：每 cycle 读出 {input-select, valid}，直接配置 crossbar。
  • Input 仅 skid register（depth 2），无 VC allocator / SW allocator。
  • 离线 schedule 保证每 link ≤1 flit/cycle → 严格无冲突、无阻塞。

3.2 Packet-switched router

  • 经典 input-queued VC router：16 VC × 8 deep SRAM / port。
  • VC allocator + SW allocator（iSLIP）+ credit-based flow control。
  • Buffer 吸收 contention → 支持任意目的地 all-to-all / any-to-any。

3.3 Hybrid router

  • 共享 crossbar + output stage；并行 slot-table 与 VC buffer 路径。
  • Mode mux：calendar window → bypass latch；packet window → buffered+allocated path。
  • Grant arbiter：reserved slot 时 calendar grant 严格优先；unreserved slot 由 SW allocator 填充。
  • 面积 ≈ packet + slot-table + mux/arb（相对 packet +5.1%）。

6.3 可行性与权衡

  • Packet → TDM 边界：packet 流量进入 TDM 区域需 admission control（AFIFO 或 reserved transit slot），否则破坏无冲突保证。
  • Any-to-any 穿越 TDM 岛：TDM-only router 无法服务未调度流 → 需空间分区或绕路（escape VC）。
  • Collective-dominated：异构 NoC 显著节省面积/功耗（本例 interior 方案 −13%），TDM backbone 精确命中 bisection 下界。
  • Global unpredictable any-to-any：需 all-packet 或 all-hybrid；异构仅在有明确 traffic 分区时最优。
  • Crossover：当 any-to-any 流量 > ~30% 总带宽时，all-hybrid 或 all-packet 更优；collective-only 场景 all-TDM 最优。

8. 与现有 CollectivePowerModule 对照

本报告 analytical model 基于同一 Orion 32nm 公式。 现有 C++ 仿真（src/collective_power.cpp）对 12×16 allgather M=16 给出 ≈66 mm² / 2935 W（含 calendar table + fork/reduce）。 本报告 16×16 all-TDM router-only 3.320 mm² 量级一致 （channel 与 fork 单元另计）。
