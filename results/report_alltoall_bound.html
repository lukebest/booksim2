All-to-All 集合通信：理论下界与 buffer 方案对比

12×16 mesh（N=192），H=4 cy / V=8 cy，ramp=1 flit/cycle， 每有向 link 1 flit/cycle。生成脚本 utils/gen_alltoall_bound_report.py。

1. 模型与参数

拓扑12×16 二维 mesh，共 192 节点。

链路水平 H=4 cy，垂直 V=8 cy；PE↔router ramp 1 cy。

二分容量 \(B = \max(\frac{X}{2} V,\ \frac{Y}{2} H) = \max(6\times8,\ 8\times4) = 48\) flit/cycle。

直径mesh 直径延迟 164 cy；含 ramp 后 drain = 166 cy。

2. 理论下界

All-to-All 瓶颈在二分带宽，不是直径：

\[ \mathrm{MK} \ge \left\lceil \frac{N(N-1)M}{B} \right\rceil + \mathrm{diameter} + 2\times\mathrm{ramp} - M \]

带宽项（大 M 主导）：\(\lceil 192 \times 191 \times M / 48 \rceil = 764 \times M\) cy。

文档中简化的 768×M 按未加权切链路数估算；本报告采用加权精确值 764×M。

3. 两种实现方案

3.1 无阻塞 · 无冲突 · 无 router buffer（W=0 刚性 pack）

  • ring_buf=0、eject_buf=0：flit 不在 router 排队，路径注入后刚性逐 hop 推进。
  • 冲突退回源 PE 注入队列；全局 offline link slot packing（边着色）。
  • 路由：XY / YX 维序最短路径单播（无 fork）。
  • M=1 机制推断 ~1068 cy（≈+15%）；M=5 按 +10%–30% 外推 4379–5175 cy。 尚无 all-to-all 专用 W=0 实测。

3.2 分组交换 + bufferable router（calendar 二分槽流水线）

  • 两阶段维序（Phase-X → Phase-Y）或等价二分槽注入（start = slot / B）。
  • Router 允许缓冲（W≥2；calendar 中 ReserveLink 冲突时等待，等价 W=∞）。
  • M=1 calendar 仿真 929 cy，精确命中下界；M=5 公式推导 3981 cy。

5. 核心结论

  1. 理论下界不可突破：无论哪种方案，makespan 受 764×M 二分带宽墙约束；加深 buffer 不能低于此下界。
  2. Bufferable 方案命中下界：calendar 二分槽流水线 M=1 实测 929 cy（1.00×）；M=5 推导 3981 cy（1.00×）；M≥16 时 makespan = 764×M，eff=1.0。
  3. W=0 有小但固定调度开销：约 10%–30%（M=1 约 1.15×），主要来自 XY 转角对齐；W≥2 后快速饱和至下界。
  4. All-to-All 对 buffer 不敏感：无多播 fork / eject 强耦合；buffer 主要用于转角 1–2 cycle 对齐，W≥2 即足够。对比 AllGather W=0 惩罚 ~2.8×。
