用 CP-SAT 生成 multi-tree AllGather 调度

0-buffer（E=0）· 无阻塞（bounded-W）· 无冲突（link / down-ramp 互斥）的双向维序多树 pattern —— 求解器流程与 ILP 对比。对应代码 utils/sched_ilp.py。

1. CP-SAT 与 ILP 的区别

两者都求解“整数变量上的优化问题”，但建模语言和求解引擎不同。

ILP / MILP（整数线性规划）CP-SAT（约束规划 + SAT）
变量 整数 + 连续混合（MILP） 仅有限域整数 / 布尔（无连续变量）
约束形式 必须是线性不等式 Ax ≤ b。组合约束要手工线性化 原生支持全局约束：AllDifferent、Cumulative、NoOverlap、reified（条件）约束，也接受线性约束
求解引擎 LP 松弛 + 分支定界 + 割平面（单纯形 / 内点） 约束传播（域过滤）+ 惰性子句学习（CDCL/SAT）+ LP 松弛 混合；多策略 portfolio 并行
表达 “一组值互不相同” 需展开成 O(n²) 个 big-M 成对不等式，或 one-hot 指派矩阵 → 模型膨胀 一行 AddAllDifferent(vars)，并有专用传播器，剪枝强
擅长问题 网络流、配比、连续资源的成本最优化 调度、排程、资源互斥这类强组合 / 析取问题

关系：CP-SAT 也能直接吃线性约束，可视为整数问题上 ILP 的超集求解器。本问题的核心约束是“同一链路 / 同一 down-ramp 的占用 cycle 互不相同”——典型的析取互斥，用 AllDifferent 表达最自然，所以选 CP-SAT 而非纯 ILP。

本模型为什么是 CP 而不是纯线性

目标 min makespan 与因果约束 a[s,c] ≥ a[s,p]+lat 都是线性的；但“每条物理链路每 cycle 至多 1 个 flit”“每个 down-ramp E=0 时每 cycle 至多 1 次 eject”本质是互斥（all-different）。在 ILP 里要靠大量 0/1 变量 + big-M 线性化，规模爆炸；CP-SAT 用全局约束 + 传播直接处理。down-ramp 带宽 B>1 时进一步用 Cumulative（容量 B 的累积资源）表达，这在纯 ILP 中也很笨重。

2. 目标 pattern 与三条硬约束

目标：在 mesh 上为 AllGather 生成一份全局发包日历——每个源 s 的 flit 沿其 X-then-Y 维序 fork 树（tree_children，覆盖全部 N 个节点）向所有节点传播，router 在分叉处复制。求 makespan 最小且物理可实现的发送时刻表。

约束物理含义模型表达
E = 00-buffer flit 到达目的节点的那一 cycle 必须被 PE 取走，down-ramp 无 eject 队列 eject cycle == arrival cycle（直接用 a[s,d] 当 eject 时刻）
bounded W无阻塞 flit 在输出端口最多滞留 W 拍即须发出；无限滞留会“踩踏”后续 flit（堵住 wormhole 流） a[s,c] ≤ a[s,p]+lat+W
conflict-free无冲突 每条有向链路、每个 down-ramp 同一 cycle 至多 1 个 flit（B>1 时至多 B 个） 链路 / eject 组内 AllDifferent（或 Cumulative）

争用不在网内堆积：若某跳在 W 内放不下，就把整个源的注入时刻整体后移（争用回退到源 PE 注入队列，那是允许排队的地方）。这正是 warmstart 贪心 greedy_arrivals 的做法。

3. CP-SAT 形式化（逐条对应代码）

决策变量

a[s, d] ∈ [0, horizon]　　源 s 的 flit 到达节点 d 的 cycle（共 N² 个整数变量）

makespan = maxs,d a[s,d] + RAMP。

约束

约束（数学）代码作用
a[s,s] ≥ RAMP model.Add(a[(s,s)]>=ramp) 源在 up-ramp 注入，注入偏移 ≥ 0
a[s,p]+lat ≤ a[s,c]
a[s,c] ≤ a[s,p]+lat+W		 model.Add(a[(s,c)]>=a[(s,p)]+lat)
model.Add(a[(s,c)]<=a[(s,p)]+lat+wcap)		 树边因果（不能未到先发）+ 有界网内等待（无阻塞）。发送时刻 = a[s,c]-lat
∀ 物理有向链路 (p,c):
AllDifferent({a[s,c]})		 model.AddAllDifferent(link_users[(p,c)]) 链路无冲突：lat 在一条链路上是常数，到达互异 ⟺ 发送 cycle 互异 ⟺ ≤1 flit/cycle
∀ 节点 d:
B=1: AllDifferent({a[s,d]})
B>1: Cumulative(demand 1, cap B)		 AddAllDifferent(grp) /
AddCumulative(iv,[1..],bw)		 down-ramp E=0：B=1 每 cycle 至多 1 次 eject；B>1 每 cycle 至多 B 次
（可选）serialize-fork
∀ 节点 p: AllDifferent(送出 cycle)		 --serfork → AddAllDifferent(node_sends[p]) 每个 router 每 cycle 至多转发 1 个 flit（fan-out ≤ 1 端口）

目标

mk = maxs,d a[s,d]；　minimize mk　（报告 makespan = mk + RAMP）
mk = model.NewIntVar(0, horizon, "makespan")
model.AddMaxEquality(mk, [a[(s,d)] for s in range(n) for d in range(n)])
model.Minimize(mk)

4. 算法流程（pipeline）

各步说明

  1. 建树 build_trees：对每个源 s，用 tree_children 生成 X-then-Y 维序双向 fork 树（先沿行铺满，再每列向上下铺满），决定每个源的 flit 走哪些物理有向链路、父子关系。
  2. 建变量：为每对 (s,d) 建整数变量 a[s,d] ∈ [0,horizon]。horizon 默认 3×LB*，越紧域越小、求解越快。
  3. 加约束：注入、因果+有界 W、链路 AllDifferent、down-ramp AllDifferent/Cumulative，可选 serialize-fork。同时把每条树边的 a[s,c] 登记进 link_users[(p,c)] 以便成组互斥。
  4. warmstart（可选） greedy_arrivals：用“离中心最远的源优先”贪心地放每个源（放不下就整体后移注入），得到一份可行解；作为 AddHint 喂给求解器，并把 horizon 收紧到贪心 makespan。大网格（如 16×16）非常依赖它。
  5. 求解 CpSolver：设 max_time_in_seconds 与 num_search_workers（多策略并行）。引擎做约束传播（AllDifferent/Cumulative 域过滤）+ 惰性子句学习 + LP 下界。返回 OPTIMAL（证明最优）/ FEASIBLE（最好可行 + 证下界）/ INFEASIBLE。
  6. 校验 _verify：用解出的值独立复查——每条链路到达互异、每节点 eject ≤ B、（serfork 时）每节点转发 cycle 互异。返回 verified=True 才采信。
  7. 抽取 pattern：发送日历 send(s, p→c) = a[s,c] − lat(p,c)，eject 时刻 = a[s,c]。这份 per-link 发包表就是最终的 multi-tree pattern（可视化 / 仿真直接用它）。

5. 用纯 ILP 生成同一 pattern 的流程

同一个问题也能写成纯线性整数规划。关键技巧：把“第几 cycle”从变量的值改成一个 0/1 开关的下标（时间索引法）。这样“每 cycle 至多几个”就变成一条简单的求和不等式，全程线性、无需 big-M。

编码对比（一句话抓住区别）

CP-SAT（值编码）纯 ILP（时间索引 / one-hot）
核心变量a[s,d] = 到达 cycle（整数本身就是时刻）x[s,d,t]∈{0,1} = 源 s 是否在第 t cycle 到达节点 d
“无冲突”AllDifferent（全局约束）每 cycle 求和 Σ ≤ 1（线性不等式）
时间轴隐式，域 [0,horizon]显式离散 t = 0..T

步骤（简明版）

  1. 建树：和 CP-SAT 完全一样——每个源建 X-then-Y 维序 fork 树，确定每个 flit 经过哪些物理链路、父子关系。
  2. 定时间轴：取上限 T（如 3×LB*），把时间离散成 t = 0,1,…,T。
  3. 定 0/1 变量：x[s,d,t] = 1 表示“源 s 的 flit 在第 t cycle 到达并 eject 于节点 d”。
    因为 x 直接代表“到达即被取走”，0-buffer(E=0) 天然成立，不用额外约束。
  4. 每个 flit 只有一个到达时刻：
    Σt x[s,d,t] = 1，　并令到达 cycle　a[s,d] = Σt t·x[s,d,t]
  5. 因果 + 无阻塞（有界等待 W）：父到了才能往子发，且最多等 W 拍：
    a[s,p] + lat ≤ a[s,c] ≤ a[s,p] + lat + W
  6. 链路无冲突：同一条物理有向链路 (p,c)，任一 cycle 至多 1 个 flit 通过：
    ∀t:　Σs 用 (p,c) x[s, c, t+lat] ≤ 1
  7. down-ramp 无冲突 / 带宽 B：每个节点 d、每个 cycle t，至多 B 次 eject（B=1 即严格 0-buffer 单端口）：
    ∀d,t:　Σs≠d x[s, d, t] ≤ B
  8. 目标：最小化 makespan（线性）：
    min mk，　s.t. mk ≥ a[s,d] ∀(s,d)
  9. 求解 + 还原：交给 ILP 求解器（单纯形 LP 松弛 + 分支定界 + 割平面）。解出的 x 还原发送日历 send(s,p→c) = a[s,c] − lat，即最终 multi-tree pattern。

第 4、6、7 步把“无冲突 / 0-buffer / 限带宽”全写成了简单求和 ≤ 常数，这正是时间索引 ILP 直观的地方。代价：变量数 ≈ N²×T，比 CP-SAT 的值编码多一个时间维，大网格上 LP 矩阵很大、求解更慢——这也是工程上选 CP-SAT 的原因。
另一种 B=1 写法：不离散时间，改用成对顺序变量 y[i,j]∈{0,1} + big-M 把“两 flit 不同 cycle”写成 a_i−a_j ≥ 1−M·y / a_j−a_i ≥ 1−M·(1−y)（经典析取调度法）。更紧凑但 big-M 让 LP 松弛很松、分支更多。

6. 典型结果与下界

两个与 buffer 无关的下界：latency_floor（最远角维序传播 + 2×ramp，物理硬墙）与 LB*（瓶颈节点 down-ramp 的 release-packing，带 B）。CP-SAT 在两者之间逼近真实最优。

配置latency_floorLB*CP-SAT 结果状态
4×4, H=4 V=6, B=2, serfork323232（W≥3；W≤2 不可行）已证最优
8×8, lat=4, B=2585867（W≥2）已证最优
16×16, lat=4, B=1122263298（W=∞, 600s）区间 [263, 298]
16×16, lat=4, B=2122143301（W=16）区间，仅可达

小网格（≤64 节点）CP-SAT 能证明最优；256 节点 ≈ 6.5 万变量，150–600s 内通常只能给 [证下界, 最好可达] 区间——这也是 ILP/CP 在强组合调度上的规模天花板。serialize-fork 不抬高最优 makespan，但会抬高所需最小 buffer 深度 W（4×4 上从 W=0 抬到 W≥3）。

7. 复现命令

# 4×4，serialize-fork，扫 W（小网格可证明最优）
../.venv-ilp/bin/python sched_ilp.py --mx 4 --my 4 --h 4 --v 6 --bw 2 --serfork --w 3

# 16×16，B=1，贪心 warmstart + 长时限
../.venv-ilp/bin/python sched_ilp.py --mx 16 --my 16 --h 4 --v 4 --bw 1 --warmstart --time 600

# 关键开关
#   --bw B      down-ramp 每 cycle eject 上限（B=1 即严格 0-buffer 单端口）
#   --w  W      每跳网内等待上限（无阻塞；缺省=无界）
#   --serfork   每 router 每 cycle 至多转发 1 个 flit
#   --warmstart 贪心解作 hint 并收紧 horizon（大网格必备）
